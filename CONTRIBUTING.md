# Contributing

Contributions are welcome. Please open an issue before submitting a pull request so the change can be discussed first. Keep pull requests focused — one feature or fix per PR. All code should pass existing tests (`pytest`) and follow the project's existing code style. For new provider support, add a sample CSV in `examples/`, extend the detector in `ai_cost_lens/providers/detector.py`, and include tests covering detection and normalization. By submitting a pull request you agree to license your contribution under the project's MIT License.
